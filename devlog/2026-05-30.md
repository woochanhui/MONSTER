# 개발 로그 - 2026년 5월 30일

## 1. 로그인 시스템 전면 개편
- **기존:** 화면 잘림 및 레이아웃 찌그러짐 현상이 있던 모달 방식.
- **변경:** 브라우저 팝업창 방식 적용.
- **상세:**
    - `Login.tsx` 독립 페이지 구현.
    - `postMessage` API를 활용한 팝업창-부모창 간의 실시간 인증 데이터(`user`, `token`) 동기화.
    - 화면 크기에 영향받지 않는 안정적인 로그인 환경 구축.

## 2. 검색 및 카테고리 필터링 구현
- **백엔드:** `Theme` 모델에 `category` 필드 추가 및 샘플 데이터 업데이트 (`modern`, `minimal`, `cyber`).
- **프론트엔드:**
    - 메인 히어로 섹션에 실시간 검색 바 추가.
    - 테마 이름 및 설명 기반의 검색 로직 구현.
    - 카테고리 버튼 클릭 시 동적 필터링 기능 활성화.

## 3. 구매 내역 영구 저장 시스템 (Persistence)
- **DB 연동:** `Purchase` 엔티티 및 `PurchaseRepository` 생성.
- **기능:**
    - 결제 성공 시 서버 DB에 구매 기록(사용자 ID, 테마 ID, 라이선스 키) 영구 저장.
    - 로그인 시 사용자의 이전 구매 내역을 자동으로 동기화하여 '내 라이브러리' 복구.
    - 새로고침 시에도 소유한 테마 데이터가 유지되도록 `fetchPurchased` 로직 연동.

## 4. 실시간 미리보기 고도화
- **테마별 프리셋:** 각 테마의 ID에 맞춰 색상, 글꼴, 레이아웃이 자동 설정되도록 개선.
- **다크 모드 최적화:** 테마별 라이트/다크 스타일을 개별 정의하고, 실시간 토글 시 모든 요소가 즉각 반응하도록 수정.
- **데이터 필드 동기화:** 백엔드(`title`, `imageUrl`)와 프론트엔드(`name`, `thumbnailUrl`) 간의 필드명 불일치 문제 해결.
